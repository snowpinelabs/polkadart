# Native libraries

This directory holds the compiled native `sr25519` library that the **Rust backend** loads via FFI,
plus the C header (`sr25519.h`) generated from the Rust source.

The pure-Dart backend needs none of this — it has no native dependency. The native library only
powers the audited [`schnorrkel`](https://docs.rs/schnorrkel) backend (the same implementation
`sp_core::sr25519` wraps).

## Files

- `sr25519.h` — C header, regenerated by `rust/build.rs` (cbindgen) on every `cargo build`.
- `<platform>/lib sr25519.<ext>` — the compiled library, produced by `tool/build_rust.sh`:
  - **Linux/Android**: `linux/libsr25519.so`
  - **macOS/iOS**: `macos/libsr25519.dylib`
  - **Windows**: `windows/sr25519.dll`

Compiled libraries are **git-ignored** — they are either built locally (`tool/build_rust.sh`) or
downloaded as signed prebuilts (`dart run sr25519:setup`). See `../BUILD.md`.

## How the loader finds the library

`lib/src/backend/platform.dart` searches, in order:

1. the per-user cache populated by `dart run sr25519:setup` (signed prebuilt download);
2. a local `rust/target/{release,debug}/` build;
3. conventional locations next to the package (`native/<platform>/`, etc.);
4. the system library path.

If none resolve, the Rust backend is simply unavailable and callers fall back to the pure-Dart
implementation automatically.
